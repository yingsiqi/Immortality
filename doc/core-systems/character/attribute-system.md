## §1 灵根体系

### 1.1 灵根纯度分配

创角时获得 **100 点灵根纯度** 自由分配于五行属性（金、木、水、火、土），单属性上限 **90 点**（纯天灵根 100 点需通过转世/特殊事件达成）。

| 灵根类型 | 判定阈值 | 纯度特征 | 创角概率 |
|----------|----------|----------|----------|
| 天灵根 | 单属性 ≥ 90 | 90+ 单极纯度 | 自由分配可达 |
| 真灵根 | 双属性合计 ≥ 100，单属性 ≥ 30 | 双属性优势 | 自由分配可达 |
| 伪灵根 | 各属性 15~25 | 均匀低纯度 | 自由分配可达 |
| 异灵根 | 特殊触发 | 五行异变融合 | 5% 随机或事件 |

#### 天灵根（单属性 ≥ 90）

- **主属性技能学习速度 +100%**，威力 **+25%**
- **无法学习** 克制属性技能（如金天灵根无法学习火系技能）
- 修炼速度同属性功法极快，异属性功法效率 0.3x

#### 真灵根（双属性 60+40 等组合）

- 双属性均有加成，学习速度 +50%，威力 +15%
- 灵活多变，无极致加成
- 可兼修两种属性功法

#### 伪灵根（各约 20）

- **所有属性效率 0.5x**（学习速度与威力）
- 后期可修炼 **大五行混元功** 弯道超车：
  - 将五行齐全的劣势转化为优势
  - 大乘期后五属性同步修炼效率反超天灵根

#### 异灵根（雷/冰/风/暗）

由两种五行属性异变融合而成，创角时有 **5% 概率** 触发，或通过游戏内特殊事件获得。

| 异灵根 | 融合源 | 特性 |
|--------|--------|------|
| 雷灵根 | 金+木 | 爆发伤害 +30%，技能有 10% 概率触发麻痹 |
| 冰灵根 | 水+金 | 控制效果 +25%，冰冻持续时间 +50% |
| 风灵根 | 木+火 | 出手速度 +20%，技能有 15% 概率触发连击 |
| 暗灵根 | 水+火（变异） | 吸血 +15%，夜战全属性 +10% |

### 1.2 灵根影响范围

| 影响维度 | 灵根作用 | 公式参考 |
|----------|----------|----------|
| 技能学习 | 同属性学习速度 × 灵根效率系数 | `学习速度 = 基础速度 × (1 + 纯度 / 100 × 加成倍率)` |
| 技能威力 | 同属性技能伤害 × 灵根威力系数 | `伤害加成 = 基础 × (1 + 纯度 × 0.0025)` |
| 初始技能 | 根据最高纯度属性赠送 1 阶对应技能 | — |
| 五行抗性 | 属性纯度 × 0.3 = 对应抗性百分比 | 抗性上限 50%（90 纯度 × 0.3 = 27%） |
| 五行相克 | PVE：克制方 ×1.25，被克制方 ×0.75；PVP：克制方 ×1.30，被克制方 ×0.70 | 相克循环：金→木→土→水→火→金 |

```python
# 灵根效率计算
def spirit_root_efficiency(purity, affinity_type, skill_element):
    """计算灵根对技能的影响"""
    if affinity_type == "heaven" and purity >= 90:
        base = 2.0  # 天灵根 2x
    elif affinity_type == "true":
        base = 1.5  # 真灵根 1.5x
    elif affinity_type == "pseudo":
        base = 0.5  # 伪灵根 0.5x
    else:
        base = 1.0

    # 异灵根特殊处理
    if affinity_type == "deviant":
        base = 1.8  # 异灵根本源属性 1.8x

    return base
```

## §2 五维属性

角色五大基础属性，通过战斗面板和各系统间接影响游戏表现，另含一项隐藏属性。

### 2.1 根骨（Physique）

| 项目 | 说明 |
|------|------|
| 核心影响 | 气血上限、体修杀伤、物理防御、负重 |
| 境界关联 | 线性增长，保障基础生存能力 |
| 体修依赖 | 体修流派的核心属性，每 10 点根骨额外 +3% 体术伤害 |

### 2.2 悟性（Comprehension）

| 项目 | 说明 |
|------|------|
| 核心影响 | 术法威能、真元上限、功法学习速度、突破瓶颈减免 |
| 境界关联 | 大境界突破时显著提升 |
| 法修依赖 | 法修流派的核心属性，影响法术暴击倍率 |

### 2.3 神识（Divine Sense）

| 项目 | 说明 |
|------|------|
| 核心影响 | 真元上限、法宝操控数量与威力、灵宠携带数量、命中率 |
| 法宝关联 | 每 20 点神识可多操控 1 件法宝，每件法宝威力 +2% |
| 灵宠关联 | 每 30 点神识可多携带 1 只灵宠 |

### 2.4 身法（Agility）

| 项目 | 说明 |
|------|------|
| 核心影响 | 移动速度、出手速度、闪避率 |
| 战场价值 | 先手判定基础值，身法高者优先行动 |
| 上限约束 | 闪避率上限 75% |

### 2.5 机缘（Fortune）

| 项目 | 说明 |
|------|------|
| 核心影响 | 会心一击（暴击率）、会心倍率（暴击伤害）、掉落加成（上限 2x）、奇遇走向 |
| 奇遇判定 | 影响特殊事件触发概率与走向分支 |
| 掉落上限 | 掉落加成最高 2 倍，不参与伤害计算 |

### 2.6 隐藏属性：毅力（Willpower）

| 项目 | 说明 |
|------|------|
| 核心影响 | 异常状态抗性、突破失败保留修为比例 |
| 获取方式 | 事件/奇遇/天资分配 |
| 异常抵抗 | 每 10 点毅力 +5% 异常状态抗性 |

### 3.2 天资点获取

每升 **1 级** 获得 **3 点可分配天资**，总计：

$$ 3 \times 83 = 249 \text{ 点天资} $$

天资用于提升§2 五维属性（§4 消耗规则），总等级满级时共 249 点。

### 3.3 境界显示规则

| 场景 | 显示内容 | 规则 |
|------|----------|------|
| 公开信息 | 大境界（如"金丹期"） | 仅显示大境界名称，不透露具体层数 |
| 探查信息 | 大境界 + 小阶段（如"金丹后期"） | 需专用探查技能/法宝，高于自身一个大境界则无法探查 |
| 自己可见 | 完整等级（如"金丹期·Lv.29·中期"） | 玩家面板完整显示 |

### 3.4 突破机制

#### 小阶段突破（前期 → 中期 → 后期）

| 项目 | 说明 |
|------|------|
| 触发方式 | 手动触发 |
| 条件 | 等级达到 + 对应丹药 + 简短突破任务 |
| 失败惩罚 | **无** |
| 流程 | 消耗丹药 → 执行任务 → 属性微调 → 阶段提升 |

#### 圆满突破（后期 → 圆满）

| 项目 | 说明 |
|------|------|
| 触发方式 | 手动触发 |
| 条件 | 等级达到后期圆满 + 丹药 + 突破任务 + **心魔判定** |
| 失败惩罚 | **损失 10% 当前修为** |
| 心魔判定系数 | `心魔强度 = f(毅力, 业力, 当前境界)` |

#### 大境界突破（圆满 → 下一大境界）

| 项目 | 说明 |
|------|------|
| 触发方式 | 手动触发 |
| 条件 | 圆满等级 + 渡劫丹药 + 渡劫任务 + **心魔判定** + **天劫** |
| 失败惩罚 | **损失 20% 当前修为** + 持续 debuff（虚弱状态 7 天） |
| debuff 效果 | 全属性 -20%，修炼效率 -30% |

```python
# 突破流程抽象
def breakthrough(player, target_realm):
    """大境界突破核心流程"""
    if is_small_stage(player, target_realm):
        # 小阶段：无惩罚
        return apply_stage_upgrade(player)
    elif is_perfect_stage(player):
        # 后期→圆满：心魔判定
        if heart_demon_check(player):
            return apply_perfect_upgrade(player)
        else:
            player.cultivation_exp *= 0.9  # 损失 10% 修为
            return "心魔阻挠，突破失败"
    elif is_realm_breakthrough(player):
        # 大境界突破：渡劫+心魔
        tribulation_result = tribulation_check(player)
        heart_demon_result = heart_demon_check(player)
        if tribulation_result and heart_demon_result:
            return apply_realm_upgrade(player)
        else:
            player.cultivation_exp *= 0.8
            apply_debuff(player, "虚弱", 7, {"all_stats": -0.2, "cultivate_speed": -0.3})
            return "渡劫失败"
```

## §4 RO 式属性消耗表

### 4.1 天资消耗规则

将天资点分配至五维属性时，消耗随属性值递增，采用分段成本模型：

| 属性值区间 | 每点消耗天资 | 累计消耗（达到该区间上限） |
|-----------|-------------|--------------------------|
| 0 ~ 20 | 1 | 20 |
| 21 ~ 40 | 2 | 20 + 20×2 = 60 |
| 41 ~ 60 | 3 | 60 + 20×3 = 120 |
| 61 ~ 80 | 4 | 120 + 20×4 = 200 |
| 81 ~ 99 | 5 | 200 + 19×5 = 295 |
| 100+ | 6 | 仅转世后可达 |

**核心设计说明**：属性本身对伤害/效果的贡献是 **线性** 的，消耗递增仅抑制极端加点，不会出现"花了很多点但伤害没涨"的情况。

## §5 战斗面板换算

所有战斗面板属性由五维属性、境界、装备共同决定。以下公式为参考值，**精确调参由战斗系统层负责**。

### 5.1 气血上限

```python
def calc_max_hp(physique, realm_level):
    """气血上限 = 根骨 × 境界系数 × 基础倍率"""
    base = physique * 20
    realm_multiplier = 1 + (realm_level - 1) * 0.35
    return int(base * realm_multiplier)
```

### 5.2 真元上限

```python
def calc_max_mp(divine_sense, comprehension, realm_level):
    """真元上限 = (神识 + 悟性×0.6) × 境界系数 × 基础倍率"""
    base = (divine_sense + comprehension * 0.6) * 15
    realm_multiplier = 1 + (realm_level - 1) * 0.4
    return int(base * realm_multiplier)
```

### 5.3 体修杀伤

```python
def calc_physique_damage(physique, realm_level):
    """体修杀伤 = 根骨 × 境界系数 × 体修倍率"""
    base = physique * 8
    realm_multiplier = 1 + (realm_level - 1) * 0.3
    return int(base * realm_multiplier)
```

### 5.4 术法威能

```python
def calc_spell_power(comprehension, realm_level):
    """术法威能 = 悟性 × 境界系数 × 法修倍率"""
    base = comprehension * 10
    realm_multiplier = 1 + (realm_level - 1) * 0.45
    return int(base * realm_multiplier)
```

### 5.5 肉身强横度

```python
def calc_flesh_toughness(physique, equipment_bonus):
    """肉身强横度 = 根骨 × 10 + 装备加成"""
    return physique * 10 + equipment_bonus
```

### 5.6 护体罡气

```python
def calc_body_shield(max_mp, comprehension):
    """护体罡气 = 真元上限 × 0.3 + 悟性 × 5"""
    return int(max_mp * 0.3 + comprehension * 5)
```

### 5.7 神识锁定（命中率）

```python
def calc_hit_rate(divine_sense):
    """命中率 = min(0.95, 0.5 + 神识 × 0.005)"""
    return min(0.95, 0.5 + divine_sense * 0.005)
```

### 5.8 迷踪幻影（闪避率）

```python
def calc_dodge_rate(agility):
    """闪避率 = min(0.75, 0.05 + 身法 × 0.007)"""
    return min(0.75, 0.05 + agility * 0.007)
```

### 5.9 会心一击（暴击率）

```python
def calc_crit_rate(fortune):
    """暴击率 = min(0.50, 0.02 + 机缘 × 0.004)"""
    return min(0.50, 0.02 + fortune * 0.004)
```

### 5.10 会心倍率（暴击伤害）

```python
def calc_crit_damage(fortune, comprehension):
    """暴击伤害倍率 = 1.5 + 机缘 × 0.01 + 悟性 × 0.005"""
    return 1.5 + fortune * 0.01 + comprehension * 0.005
```

## §6 跨境界数值碾压机制

### 6.1 设计目的

83 级加点体系刻画的是**同境界内**的属性差异（天才炼气 vs 普通炼气）。跨境界的质变碾压必须由独立机制实现——境界系数。

### 6.2 境界系数

每个大境界对应一个境界系数 `2^n`（n = 境界序号，炼气起算）：

| 境界 | 序号 n | 境界系数（2^n） | 备注 |
|------|--------|:----:|------|
| 炼气 | 0 | **1x** | 基准 |
| 筑基 | 1 | **2x** | 洗髓伐脉，凡体蜕仙胎 |
| 金丹 | 2 | **4x** | 金丹凝结，法力固化 |
| 元婴 | 3 | **8x** | 元婴诞生，神魂离体 |
| 化神 | 4 | **16x** | 化神入道，触及法则 |
| 炼虚 | 5 | **32x** | 炼虚返实，空间洞穿 |
| 合体 | 6 | **64x** | 身合天地，世界投影 |
| 大乘 | 7 | **128x** | 飞升前夕，天道共鸣 |

**待定选项**：翻倍基数是否用 `e ≈ 2.718`（增速更陡，大乘 ≈ 1096x，碾压更强但数值跨度更大）。当前默认 2^n。

### 6.3 应用于所有战斗面板

境界系数不是只作用于血蓝，而是**所有战斗面板的最终乘法因子**：

```
最终面板值 = 境界系数 × 属性计算值

### 6.5 为什么不做更多机制

分身、出窍、领域、法则等能力不放在境界系数中，而是通过**术法/功法的学习要求**来限制：

```

```

功法/术法系统天然承担了"境界解锁新能力维度"的职责，境界系数只负责纯粹的数值碾压。

def attribute_warning(player):
    """属性预警系统"""
    threshold_map = {
        "QiRefining":  {"physique": 5,  "comprehension": 5,  "divine_sense": 5,  "agility": 5,  "fortune": 5},
        "Foundation":  {"physique": 15, "comprehension": 12, "divine_sense": 12, "agility": 10, "fortune": 8},

        "Divine":      {"physique": 120,"comprehension": 90, "divine_sense": 95, "agility": 65, "fortune": 55},
        "Void":        {"physique": 200,"comprehension": 150,"divine_sense": 160,"agility": 100,"fortune": 85},
        "Unity":       {"physique": 350,"comprehension": 240,"divine_sense": 280,"agility": 160,"fortune": 130},
        "Ascension":   {"physique": 550,"comprehension": 380,"divine_sense": 450,"agility": 250,"fortune": 200},
    }

    warnings = []
    realm_key = player.realm_name
    if realm_key in threshold_map:
        thresholds = threshold_map[realm_key]
        attrs = {
            "physique": player.physique,
            "comprehension": player.comprehension,
            "divine_sense": player.divine_sense,
            "agility": player.agility,
            "fortune": player.fortune,
        }
        for attr_name, threshold in thresholds.items():
            if attrs[attr_name] < threshold:
                ratio = attrs[attr_name] / max(threshold, 1)
                if ratio < 0.6:
                    warnings.append(f"[严重] {attr_name} 严重不足({attrs[attr_name]}/{threshold})")
                elif ratio < 0.85:
                    warnings.append(f"[警告] {attr_name} 偏低({attrs[attr_name]}/{threshold})")

    return warnings
```
