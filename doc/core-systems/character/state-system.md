> 本文档是角色系统拆分子文档之一，聚焦角色所有状态位的定义、触发机制、效果与叠加规则，对齐 L1 规范。状态体系覆盖正面状态、负面状态、特殊状态三大类别，以及状态叠加与清除的通用规则。

## §2 正面状态

### 2.1 顿悟

| 项目 | 说明 |
|------|------|
| 触发方式 | 悟性概率触发（悟性越高概率越大）、特殊丹药（悟道丹等）、秘境奇遇 |
| 效果期间 | 功法修炼速度 **3x**，技能熟练度 **翻倍** |
| 持续时间 | 15~120 分钟（现实时间），随触发方式浮动 |
| 顶级顿悟 | 可越阶学习技能（无视境界前置条件 1 阶） |
| 悟性加成 | 悟性 ≥ 90 时，顿悟触发概率翻倍 |
| 叠加规则 | 同类不叠加，取最长持续时间 |

```python
# 顿悟触发概率
def enlightenment_chance(comprehension, has_pill=False, in_zone=False):
    """每日首次顿悟判定"""
    base = comprehension * 0.003  # 悟性 50 时基础概率 15%
    if comprehension >= 90:
        base *= 2  # 悟性 90+ 概率翻倍
    if has_pill:
        base += 0.25  # 悟道丹 +25%
    if in_zone:
        base += 0.15  # 秘境奇遇 +15%
    return min(0.75, base)  # 上限 75%
```

### 2.2 劫余 Buff

| 项目 | 说明 |
|------|------|
| 触发方式 | 成功渡劫后自动获得 |
| 效果 | 全属性 **+10%**（包含五维属性与面板属性） |
| 突破瓶颈减免 | 突破成功率 +5%，失败修为损失 -50%（仅限小阶段突破） |
| 持续时间 | 整个境界周期，至下次渡劫时自动刷新替换 |
| 叠加规则 | 不可叠加，每次渡劫刷新持续期间与效果 |

### 2.3 涅槃印记（转世者专属）

| 项目 | 说明 |
|------|------|
| 触发方式 | 每次转世后自动获得 |
| 效果 | 技能学习速度 **+25%**，经验获取 **+25%** |
| 灵根上限 | 灵根纯度上限 **+10**（每转一次叠加一次） |
| 叠加规则 | 多转叠加，为角色永久根基（印刻于灵魂底层） |
| 驱散性 | **不可驱散**，不受驱散类技能/丹药影响 |

```python
# 涅槃印记效果叠加
def nirvana_bonus(reincarnation_count):
    """计算涅槃印记累计加成"""
    learn_speed_bonus = 0.25 * reincarnation_count
    exp_bonus = 0.25 * reincarnation_count
    spirit_root_cap = 10 * reincarnation_count
    return {
        "learn_speed_bonus": min(learn_speed_bonus, 0.75),  # 上限 75%
        "exp_bonus": min(exp_bonus, 0.75),                  # 上限 75%
        "spirit_root_cap_bonus": min(spirit_root_cap, 30),  # 最多 3 转
    }
```

## §3 负面状态

### 3.1 心魔

心魔是最危险的负面状态之一，不会自行消退，必须主动消除。

| 项目 | 说明 |
|------|------|
| 触发方式 | 突破失败、高阶术法反噬（高于自身 2 个境界以上的术法）、精神类攻击 |
| 效果 | 悟性 **-20%**，突破成功率 **-10%** |
| 修为流失 | 每分钟自动流失 **0.1%** 当前修为 |
| 持续时间 | **不会自行消退**，必须主动消除 |
| 消除方式 | 特殊丹药（清心丹、凝神丹等）、完成心魔试炼任务 |
| 叠加上限 | **3 层**，每层独立生效 |

```python
# 心魔层数效果
def heart_demon_effect(layers):
    """
    心魔效果 = 基础 × 层数（加法叠加）
    3 层时：悟性 -60%，突破成功率 -30%，修为流失 0.3%/min
    """
    layers = min(layers, 3)
    return {
        "comprehension_penalty": -0.20 * layers,
        "breakthrough_penalty": -0.10 * layers,
        "cultivation_loss_per_min": 0.001 * layers,
    }
```

### 3.2 伤势系统

伤势分为四级，按当前 HP 损失比例判定：

#### 伤势等级速查

| 伤势等级 | HP 损失 | 属性惩罚 | 恢复方式 | 自然恢复时间 |
|----------|---------|----------|----------|-------------|
| 轻伤 | < 30% | 全属性 **-10%** | 自然恢复或普通丹药 | 约 5 分钟 |
| 重伤 | 30% ~ 60% | 全属性 **-25%** | 需丹药介入，自然恢复极慢（约 2 小时） | — |
| 濒死 | > 60% | 全属性 **-50%** | 必须高级丹药 + 休养，**修为可能跌落** | 24 小时以上 |
| 灵根损伤 | 特殊 | 灵根属性临时降低 | 需天材地宝修复 | — |

#### 伤势层级详细说明

**轻伤（HP 损失 < 30%）**

| 项目 | 说明 |
|------|------|
| 属性惩罚 | 全属性 **-10%** |
| 自然恢复 | 脱战后每 30 秒恢复 5% HP，约 5 分钟完全恢复 |
| 丹药加速 | 普通回血丹可立即消除轻伤状态 |

**重伤（HP 损失 30%~60%）**

| 项目 | 说明 |
|------|------|
| 属性惩罚 | 全属性 **-25%** |
| 自然恢复 | 脱战后每 30 秒恢复 1% HP，约需 2 小时完全恢复 |
| 丹药需求 | 必须使用中级以上疗伤丹药方可加速消除 |
| 战斗力影响 | 无法使用高阶技能（高于自身 1 个境界的技能锁定） |

**濒死（HP 损失 > 60%）**

| 项目 | 说明 |
|------|------|
| 属性惩罚 | 全属性 **-50%** |
| 修为跌落风险 | 濒死状态下再次受击，有 30% 概率跌落 1 级修为（大境界内） |
| 丹药需求 | 必须使用高级疗伤丹药（九转还魂丹等） |
| 休养要求 | 至少 24 小时现实时间方可完全恢复 |
| 技能锁定 | 所有技能锁定，仅保留基础普攻与移动能力 |

**灵根损伤（特殊伤势）**

| 项目 | 说明 |
|------|------|
| 触发方式 | 特殊攻击（针对灵根的灵魂攻击）、渡劫失败（雷劫反噬灵根） |
| 效果 | 灵根属性临时降低 **30~50 点**纯度（随机），对应属性技能效率大幅下降 |
| 修复方式 | 天材地宝（九幽冥莲、五行灵髓等）或特殊事件修复 |
| 自然恢复 | **不可**自然恢复 |
| 长期影响 | 若不修复，修炼该属性功法效率持续 -50% |

```python
# 伤势 HP 阈值判定
def wound_level(hp_ratio):
    """根据当前 HP 比例判定伤势等级"""
    if hp_ratio >= 0.95:
        return None  # 健康
    elif hp_ratio >= 0.70:
        return "light"      # 轻伤
    elif hp_ratio >= 0.40:
        return "severe"     # 重伤
    else:
        return "critical"   # 濒死
```

## §4 特殊状态

### 4.1 转世清算期（仅转世后 7 天内）

转世后角色进入为期 7 天的清算期，属性与记忆逐渐恢复：

| 项目 | 说明 |
|------|------|
| 触发 | 每次转世后自动触发 |
| 持续时间 | **7 天**（现实时间，离线也计时） |
| 灵根重置 | 灵根纯度重置为 **0**，每日自动恢复 **15 点** |
| 恢复过程 | 第 1 天 15 → 第 2 天 30 → ... → 第 7 天 105（超出 100 部分转为灵根溢出值） |
| 记忆碎片 | 第一世记忆碎片分散于各地图，拾取可获得**额外天资点**（每片 +3，总计最多 +30） |
| 功法状态 | 全部功法遗忘，但保留 **涅槃印记** 的熟练度加成效果 |
| 技能保留 | 被动类技能（如体质类）保留，主动类技能全部遗忘 |

```python
# 转世清算期灵根恢复
def reincarnation_recovery(day):
    """清算期第 N 天的灵根恢复值"""
    day = min(max(day, 1), 7)
    base = day * 15  # 每日恢复 15 点
    return min(base, 100)  # 超出 100 转为溢出值
```

### 4.2 天劫预警（大境界圆满后）

角色到达大境界圆满时，系统触发天劫预警：

| 项目 | 说明 |
|------|------|
| 触发条件 | 达成大境界圆满（如金丹圆满→元婴） |
| 倒计时 | **3 天**（现实时间），到期后自动进入渡劫流程 |
| 主动触发 | 可在倒计时期间主动触发渡劫，**无额外加成** |
| 渡劫准备 | 可准备渡劫阵法（成功率 +10~25%）、渡劫丹药（成功率 +5~15%）、防御法宝（伤害减免） |
| 预警提示 | 系统界面显示倒计时与准备清单 |
| 渡劫失败后果 | 损失 **20%** 修为 + 虚弱 debuff 7 天（全属性 -20%，修炼效率 -30%） |

### 4.3 PVP 状态

PVP 状态由角色所处的当前区域自动决定，不可手动切换：

| 区域类型 | 规则 | 说明 |
|----------|------|------|
| 安全区 | **禁止 PVP** | 主城、宗门驻地、新手村等，任何攻击行为无效 |
| 中立区 | **可宣战 PVP** | 野外地图、秘境入口等，需主动宣战方可攻击，对方可拒绝（消耗道具） |
| 混乱区 | **自由 PVP** | 特定秘境、战场、血战之地等，无限制攻击 |

#### 罪恶值系统

在非混乱区击杀玩家（尤其是击杀低境界或同境界玩家）会产生罪恶值：

| 击杀对象 | 罪恶值增加 | 说明 |
|----------|-----------|------|
| 同境界玩家 | +10 | 正常击杀 |
| 低 1 个大境界 | +30 | 以大欺小 |
| 低 2+ 个大境界 | +100 | 严重以大欺小 |
| 高境界玩家 | +3 | 越级击杀，罪恶值较低 |
| NPC（非任务） | +5 | 无故击杀 NPC |

#### 罪恶值影响

| 罪恶值范围 | 影响效果 |
|-----------|----------|
| 0 | 正常状态 |
| 1~50 | NPC 商店价格 **+10%**，部分任务不可接取 |
| 51~200 | NPC 商店价格 **+30%**，宗门贡献获取 -50%，安全区 NPC 守卫主动攻击 |
| 201~500 | 全图赏金通缉，所有玩家可攻击你（混乱区外也视为可攻击目标） |
| 500+ | 天罚：雷劫每 24 小时降临一次，直至罪恶值降至 500 以下 |

```python
# 罪恶值清除
def sin_reduction(current_sin, is_meditating=False, has_item=False):
    """罪恶值自然衰减"""
    daily_reduction = 10  # 每日自然衰减
    if is_meditating:
        daily_reduction *= 2  # 打坐忏悔双倍衰减
    if has_item:
        daily_reduction += 30  # 使用清罪符额外衰减
    return max(0, current_sin - daily_reduction)
```

## §5 状态叠加规则

### 5.1 正面状态叠加

| 规则 | 说明 |
|------|------|
| 同类不叠加 | 同一种正面状态同时存在时，**取最高效果**，不叠加也不刷新持续时间 |
| 异类共存 | 不同种类正面状态可同时生效，各自独立计算 |
| 优先级 | 若效果冲突（如两状态对同一属性有增益），取增益绝对值较大的生效 |

### 5.2 负面状态叠加

| 规则 | 说明 |
|------|------|
| 同类可叠加 | 同一种负面状态可叠加，**上限 3 层** |
| 叠加方式 | 效果**加法叠加**（如心魔 1 层 -20% 悟性，3 层 -60% 悟性） |
| 异类共存 | 不同种类负面状态独立生效，效果加法叠加至面板 |
| 上限保护 | 同属性惩罚上限为 **-80%**（如悟性受心魔 + 濒死双重惩罚，叠加不超过 -80%） |

### 5.3 正负状态共存

| 规则 | 说明 |
|------|------|
| 各自独立生效 | 正面与负面状态同时存在时，互不抵消，各自独立计算 |
| 最终值计算 | `最终属性 = 基础属性 × (1 + 正面加成总和) × (1 + 负面惩罚总和)` |
| 示例 | 劫余 buff（全属性 +10%）+ 轻伤（全属性 -10%）= 最终全属性 × 1.10 × 0.90 = ×0.99（小幅净亏损） |

```python
# 正负状态混合计算
def final_attribute_value(base_value, buffs, debuffs):
    """
    最终属性值计算：
    - buffs：正面状态效果列表（如 [0.10, 0.25] 表示 +10%, +25%）
    - debuffs：负面状态效果列表（如 [-0.20, -0.10] 表示 -20%, -10%）
    """
    buff_multiplier = 1 + sum(buffs)
    debuff_multiplier = 1 + sum(debuffs)
    # 负面惩罚上限保护
    debuff_multiplier = max(debuff_multiplier, 0.20)  # 最多降低到 20% 即 -80%
    return base_value * buff_multiplier * debuff_multiplier
```

### 5.4 状态清除规则

| 场景 | 清除范围 | 保留项 |
|------|----------|--------|
| 死亡 | 清除所有非永久状态 | 保留 **涅槃印记**（转世永久绑定） |
| 转世 | 清除所有状态 | 涅槃印记转世后重新获得 |
| 大境界突破 | 清除所有持续型负面状态（心魔除外） | 心魔需单独消除 |
| 驱散技能/丹药 | 按技能/丹药描述清除对应状态 | 不可驱散类状态（如涅槃印记）不受影响 |

### 5.5 状态优先级汇总

