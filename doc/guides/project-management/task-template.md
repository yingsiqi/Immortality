# 任务模板

> 复制本模板后，可作为单个执行任务的标准输入。

```md
# TASK-ID: 任务标题

## 目标

一句话说明这个任务要解决什么问题。

## 阶段

- 所属阶段：P0 / P1 / P2 / P3 / P4
- 所属工作流：客户端 / 服务端 / 数据 / 设计 / 运维 / QA

## 背景

- 为什么要做这件事
- 它依赖哪些前置任务
- 做完后会解锁什么

## 输入上下文

- 必读文档 1：
- 必读文档 2：
- 必读规则：
- 相关代码目录：

## 范围

### 本任务要做

- 
- 
- 

### 本任务不做

- 
- 
- 

## 约束

- 不得修改哪些目录或接口
- 是否必须走 Gateway
- 是否允许引入新依赖
- 是否需要与你确认

## 验收标准

- [ ] 条件 1
- [ ] 条件 2
- [ ] 条件 3

## 验证方式

- 编译命令：
- 测试命令：
- 手工验证步骤：

## 交付物

- 修改文件：
- 新增文件：
- 更新文档：

## 风险与阻塞

- 可能风险：
- 需要确认事项：

## 完成回写

- 任务总板状态：
- 是否需要新增后续任务：
- 是否需要更新设计确认队列：
```
