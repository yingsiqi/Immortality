## 触发机制概览

| 触发方式 | 说明 | 典型事件 |
|---------|------|---------|
| 条件触发 | 满足特定条件后必然或概率触发 | 击杀精英→古修洞府；境界达标→任务链 |
| 概率触发 | 在某种行为发生后按概率判定 | 采集→灵药现世(8%)；移动→灵兽受困(2%) |
| 时间触发 | 在固定时间点或多长时间后触发 | 天道事件按日历时间；赛季事件按赛程周 |
| 行为触发 | 玩家的某种行为累计达到阈值 | 击杀妖兽超过 N 只；坊市抛售材料超过 N 个 |
| 任务前置 | 需要完成前置任务/剧情才能解锁 | 任务链的后续阶段；部分连锁事件 |

## 编写原则

- **原著对齐**：优先录入原著明确涉及的奇遇场景（古修洞府、上古修士残念、韩立之路等），标注「原著来源」或「原著扩展」
- **区域引用真实 ID**：所有触发区域引用 regions.md 中的真实 region_xxx ID
- **奖励引用真实 ID**：装备/材料/丹方/灵宠奖励引用已有数据表真实 ID
- **MMORPG 数值梯度**：事件难度与奖励覆盖全等级段（Lv1-10），形成合理的投入产出比
- **事件互链**：任务链/连锁事件之间通过 event_id 互相引用，形成事件网络
- **赛季事件轮换**：8 条赛季事件按 12 周赛程编排，对齐 L7 §9.4 赛季系统
- **留扩展空间**：每类事件不填满所有等级段，为后续版本新增事件留白
