# 团结引擎客户端规则

## 工程定位

- `game/` 是唯一客户端工程目录
- `Assets/Scripts/Core` 放管理器、系统基类和公共工具
- `Assets/Scripts/Features` 按玩法系统划分模块
- `Assets/Scripts/Network` 统一封装 HTTP 和实时通信

## 开发约束

- 优先保留最小 MVP 纵切：认证、角色、修炼、基础战斗
- 客户端状态同步优先通过网关和实时服务，不自造第二套协议
- 运行时常量优先收敛到 `GameConstants`
- 脚本目录与文档里的系统边界保持一致

## 初始化要求

- 团结引擎版本、包依赖和项目设置必须先固定后再大规模开发
- 新增第三方插件前先记录用途、来源和版本
