# 仓库结构规则

## 根目录职责

- `doc/`：策划、规则、架构和部署文档的单一事实源
- `game/`：团结引擎客户端工程根目录
- `server/`：.NET 8 服务端代码
- `deploy/`：Docker、Nginx、监控与部署配置
- `admin-panel/`：文档与开发服务管理面板

## 结构约束

- 不再在仓库中引入第二套 Web 客户端目录
- 客户端脚本、Prefab、场景和资源统一放在 `game/Assets/`
- API 契约、端口、服务说明统一先更新文档，再修改代码
- 新增模块前先判断应归属 `game/`、`server/` 还是 `tools/`

## 禁止事项

- 不要把 Unity/Tuanjie 缓存目录提交到 Git
- 不要在 `doc/` 外维护另一份架构说明
- 不要让同一功能同时存在两套客户端实现
